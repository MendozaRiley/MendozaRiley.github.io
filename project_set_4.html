Exercise 7


For this exercise, I converted my Processing code for Project 1 to p5.js. While I kept many of the same drawing tools, I added a bezier curve to the rmQuills custom function and a new custom function, rmAlien2, that employs shearX().

Custom Function's:

  • rmQuills: This is the function for the black and white lines that represent tree/grass like shapes of the alien sighting scene.
  • rmStars: This is the function for the random small white circles scattered around the sketches.
  • rmAlien1: This is the function for the alien figure that radiates light blue and purple circles from its core.
  • rmAlien2: This is the function for the alien figure made up of translucent quadrilaterals.
  • rmTopBot: This function contains the code for the vertical circles of the alien figure.
  • rmLeftRight: This function contains the code for the horizontal circles of the alien figure.
  • rmUFO1: This is the function for the first UFO shape that "comes in piece."
  • rmUFO2: This is the function for the second UFO shape with the white light emanating from the bottom.
  • rmUFO3: This is the function for the third UFO shape that emits the blue-ish green rays that will "destroy us all."
  • rmUTop: This function contains the code for the top halves of the UFOs.
  • rmUBot: This function contains the code for the bottom halves of the UFOs.

Exercise 9


For this exercise, I experimented with different libraries including p5 sound, scenemanager, p5 play, and rotating knobs by Miles De Coster. The three sprites on the home page correspond to a different scene. The home sprite button on the supernova, UFO, and about scenes lead back to the home page. I used sound effects for each sprite button, and a sound plays during the supernova scene. The rotating knobs are seen in the UFO scene where the user can alter the colors of some of the images within the sketch. Each of the sprites are my original creations that serve as representations of their scene destinations. The user can also switch between scenes by pressing the keys: 1, 2, 3, and 4.

Project 4 Proposal

I am planning to create a short story-based adventure game with two embedded mini-games that will impact the player’s ending. My project will be the first part of the storyline, which will coincide with Lily Mendoza’s project, the second part. The game will offer an outrageous storyline with characters based on celebrities and famous fictional characters in order to parody the nature of fan fiction and self-insert stories. We aim to comment on the idea of gamification by going over the top with the theatrics of the interface by including vibrant colors and an unnecessary amount of sound effects when the user interacts with the game, as well as numerous animations and interactive sprites. In addition, considering how fan fiction and self-insert games are meant to simulate an alternate reality, I am planning to use skeuomorphic design to highlight the reliance of this fictional world on the people and events of the real world

Project 4


This is the first part of a story-based adventure game titled "An MC Adventure." Part I features two embedded mini-games that impact the player's ending in Part II. The storyline coincides with Lily Mendoza's project, Part II. The game offers an outrageous storyline with characters based on celebrities and famous fictional characters in order to parody the nature of fanfiction and self-insert stories. This interface aims to comment on the idea of gamification by going over the top with the threatrics from vibrant colors to unnecessary sound effects to numerous interactive sprites. Additionally, considering how fanfiction and self-insert games are meant to simulate an alternate reality, I used more skeuomorphic design to highlight the reliance of this digital world on the people and events of the physical one. The interface includes a multitude of bitmaps and sound effects, an ungodly amount of different states, and multiple help screens for each main state. While the mini-games involve mouse driven navigation, the narratives scenes mainly use keyboard input. The three libraries I used are p5.sound, p5.scenemanager, and p5.play.
